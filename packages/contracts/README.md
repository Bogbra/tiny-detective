# Contracts

Shared API contracts between `services/api` and `apps/game`: the OpenAPI schema and generated Dart DTOs derived from it.

**Still intentionally empty — the anticipated trigger never actually happened, confirmed now that the project is well past the phase that would have populated it.** `apps/game`'s data-layer DTOs are hand-written (see `apps/game/lib/features/*/data/dtos/`, no codegen anywhere in this repo) and verified against the real backend contract via `services/api/tests/contract/` on the Python side and real end-to-end/live verification on the Flutter side — the same discipline this project uses everywhere else (see `docs/testing-strategy.md`). That turned out to be sufficient at this project's scale; generating Dart types from FastAPI's own auto-exposed OpenAPI schema (`/openapi.json`) would be real, legitimate tooling to add if the API surface grew large enough for hand-written DTOs to become error-prone or tedious to keep in sync — not needed yet, and not built speculatively ahead of that need. See task 20 of the security/ops audit and ADR-0008 (API versioning) for the same "build it when the real need shows up, not before" reasoning applied elsewhere in this project.
